## 分析三：P2-3 Bean 命名冲突排查

### 问题描述

发现 6 个 Controller 使用 `"XxxController2"` 格式的显式 Bean 命名，疑似存在 Spring 容器 Bean 命名冲突。

### 排查结果

| Controller | 显式 Bean 名 | 所在包 |
|------------|-------------|--------|
| `CateTypeController` | `"CateTypeController2"` | `grp.pt.controller.basedata` |
| `CatalogController` | `"CatalogController2"` | `grp.pt.controller.basedata` |
| `ElementController` | `"ElementController2"` | `grp.pt.controller.basedata` |
| `ElementValueController` | `"ElementValueController2"` | `grp.pt.controller.basedata` |
| `IdGenController` | `"IdGenController2"` | `grp.idgen.feignclient` |
| `IdgenManagerController` | `"IdgenManagerController2"` | `grp.idgen.feignclient` |

### 结论

- 均为 **有意设计** 的显式 Bean 命名
- 每个 Bean 名称在 Spring 容器中唯一，不存在实际冲突
- 推测原因：与其他模块（如 `grp-element-api` 中的 Feign 接口）同名类区分
- **不修改**：修改需评估全链路影响

## 分析四：P2-4 Service 层接口混放检查

### 问题描述

`grp.pt.service` 根目录保留了 6 个接口文件，疑似混放。

### 排查结果

根目录接口列表：

| 接口名 | 实现类位置 | 用途 |
|--------|-----------|------|
| `ValueControlManager` | `service/impl/ValueControlManagerImpl` | 值集控制管理器 |
| `IEleRelationService` | `service/impl/EleRelationServiceImpl` | 要素关系服务 |
| `IElementValueSetService` | `service/impl/ElementValueSetServiceImpl` | 要素值集服务 |
| `IOrgFeignService` | `service/impl/OrgFeignServiceImpl` | 组织 Feign 服务 |
| `InformationCheckService` | `service/impl/InformationCheckServiceImpl` | 信息校验服务 |
| `IProcessElementService` | `service/impl/ProcessElementServiceImpl` | 流程要素服务 |

### 结论

- 全部为 **接口定义**，实现类在 `service/impl/` 中
- 与 `service/basedata/` 子目录形成两层服务结构，属于跨域/门面接口分离设计
- **不修改**：移动将影响 `element-server` 和 `element-server-com` 两个模块的大量 import

## 不可修改项说明

以下 P2 问题因"不修改 URL / HTTP 方法 / entity 包名"的约束原则而未处理：

| 问题 | 约束原因 |
|------|----------|
| RESTful 动词不规范（如 POST 用于查询） | 修改会改变 HTTP 方法，影响前端调用 |
| Model 层 entity 包名不规范 | 涉及 entity 包名变更 |
| DAO 层非标准命名 | 下游依赖广泛 |
| DTO 类内部属性命名（下划线风格） | 序列化兼容性风险 |

---

## 修改原则

- 仅修改 `package` 声明和 `import` 语句
- 不改变 HTTP 接口 URL、HTTP 方法、入参、出参
- 不修改 entity 类包名
- 不改变任何业务逻辑
- 所有变更均添加 AI 代码标记

---

## 风险评估

| 风险项 | 评估 |
|--------|------|
| 接口兼容性 | 无影响，所有 `@RequestMapping` 路径不变 |
| Spring 容器 | `@ComponentScan` 扫描 `grp.pt` 及子包，新子目录自动覆盖 |
| 编译兼容 | 原文件已删除，无类重复；所有 import 已同步更新 |
| 回归范围 | Controller 层目录迁移，建议全模块编译验证 |
