# P4 变更报告：Service 层 facade/impl 接口实现分离

## 基本信息

| 项目 | 内容 |
|------|------|
| 优先级 | P4（Service 接口实现分离优化） |
| 影响模块 | `element-server-com` |
| 迁移文件总数 | 108 |
| 接口文件(facade/) | 51 个文件 |
| 实现文件(impl/) | 57 个文件 |
| import 引用更新 | 130 个文件 |
| 变更日期 | 2026-03-17 |
| 风险等级 | 中等（涉及跨模块 import 引用） |

---

## 变更概述

将 Service 层从原有的**按业务子包分散结构**（根目录、basedata/、bookset/、agencyManager/、executor/）统一重组为**接口/实现分离结构**（facade/ + impl/），遵循 SKILL.md 标准四层架构规范。

### 分类原则

| 分类依据 | 归属目录 | 说明 |
|---------|---------|------|
| Service 接口定义（I*Service、*Manager、*Service 接口） | `service/facade/` | 统一存放所有 Service 接口 |
| Service 实现类（*ServiceImpl、*Impl、具体实现） | `service/impl/` | 统一存放所有 Service 实现 |
| 非 Service 业务文件（constant、enums、model、util 等） | `service/{业务包}/` | 保留原位，不迁移 |

## 与 P0-P3 报告的关系

本 P4 报告是 P0-P3 系列结构优化的延续：

| 阶段 | 变更内容 | 说明 |
|------|---------|------|
| P0 | Controller 层依赖违规修复 | 消除 Controller→Controller 依赖 |
| P1 | 目录命名与分类规范修复 | imp→impl 命名修正，DTO 归类 |
| P2 | Controller 目录组织与结构优化 | Controller 功能子目录分组 |
| P3 | Controller 层 custom/common 接口分离 | Controller 外部/内部分离 |
| **P4** | **Service 层 facade/impl 接口实现分离** | **Service 接口/实现统一分离** |

---

## 修改原则

- 仅修改 `package` 声明和 `import` 引用，不改变任何业务逻辑
- 非 Service 文件（constant、enums、exception、feignclient、model、util）保留原位
- 所有 Service 接口统一归入 `facade/`，所有实现统一归入 `impl/`
- `@ComponentScan` 扫描 `grp.pt` 及子包，新子目录自动覆盖
- synchronization/ 作为独立业务域，其内部 Service 保留原有 facade/impl 结构

---
