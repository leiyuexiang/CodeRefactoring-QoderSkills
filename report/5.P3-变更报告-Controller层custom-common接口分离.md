## 变更概述

将 Controller 层从原有的**功能分类结构**（basedata/、api/、util/、bookset/、agencyManager/、notify/、common/）重组为**外部/内部接口分离结构**（custom/ + common/），遵循 SKILL.md 标准四层架构规范。

### 分类原则

| 分类依据 | 归属目录 | 说明 |
|---------|---------|------|
| 面向前端 UI 的业务接口 | `controller/custom/` | 外部接口，面向前端/第三方调用 |
| 内部 API / 工具 / 调试 / 同步 | `controller/common/` | 内部接口，面向内部微服务调用 |

## 与 P2 报告的关系

本 P3 报告是在 P2 变更报告基础上的**进一步结构优化**：

| 阶段 | 变更内容 | 说明 |
|------|---------|------|
| P2 | Controller 根目录文件归类到功能子目录 | basedata/、api/、util/、common/、notify/ |
| P2 | controller/imp/ 目录修正 | imp/ → basedata/ |
| **P3** | **功能子目录 → custom/common 接口分离** | **basedata/ → custom/basedata/ 等** |

P2 阶段建立的功能分组在 P3 阶段被保留为二级子目录，仅在上层增加了 custom/common 一级分类。

---

## 修改原则

- 仅修改 `package` 声明，不修改 `import` 语句（Controller 之间无交叉引用）
- 不改变 HTTP 接口 URL、HTTP 方法、入参、出参
- 不修改任何业务逻辑
- 所有变更均包含 AI 代码标记（K7P2X 为整体迁移标记）
- `@ComponentScan` 扫描 `grp.pt` 及子包，新子目录自动覆盖

---

## 风险评估

| 风险项 | 评估 |
|--------|------|
| 接口兼容性 | 无影响，所有 `@RequestMapping` 路径不变 |
| Spring 容器 | `@ComponentScan` 扫描 `grp.pt` 及子包，新子目录自动覆盖 |
| 编译兼容 | 原文件已删除，无类重复；Controller 之间无交叉 import |
| 回归范围 | Controller 层目录迁移，建议全模块编译验证 |
