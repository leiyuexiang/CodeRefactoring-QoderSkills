## 五、依赖关系图

```
                    element-demo2 (根POM)
                         │
                    element-module
                    ┌────┴────────────────────┐
                    │                         │
          grp-capability-element    grp-aggregation-element
          ┌─────┼──────────┐              │
          │     │          │    element-server-springcloud
  grp-element-api │   element-server            │
          │     │          │               依赖
          │  element-server-com ◄──────── element-server
          │     │
          └─────┘
         (API依赖)
```

**依赖方向（自上而下）：**
1. `element-server-springcloud` → `element-server`（聚合层依赖能力层）
2. `element-server` → `element-server-com`（Controller层依赖实现层）
3. `element-server-com` → `grp-element-api`（实现层依赖API模块）
4. `element-server` → `grp-element-api`（Controller层依赖API模块）

## 六、注意事项

1. **编译产物清理**：`target/classes/` 下的旧编译产物仍包含旧配置，需执行 `mvn clean compile` 消除
2. **体验层预留**：`grp-experience-element/` 已创建骨架，目前为空；后续如需 Feign SDK 可在此添加 `element-feign-com` 模块
3. **外部父POM**：原先各模块的外部父POM（`grp-platform-server`、`grp-platform-springcloud`、`grp-platform-api`）已不再被叶子模块直接引用，依赖管理需在新的根POM或容器POM中配置 `dependencyManagement`
4. **聚合层适配模块**：规范建议补充 `element-server-huawei/` 和 `element-server-tencent/` 适配模块（非强制）
5. **底座层**：本工程无独立底座层（`grp-common-boot/`），底座层由外部工程提供
