## 问题描述

`ProcessElementController` 中存在 **Controller→Controller 直接依赖**，通过 `@Autowired` 注入了 `ElementValueController`，违反了四层架构中"同层不依赖"的核心原则。

### 问题代码

```java
// 原始代码 - 违规：Controller 直接注入另一个 Controller
@Autowired
private ElementValueController elementValueController;
```

该依赖导致 `ProcessElementController` 直接调用 `ElementValueController` 的以下方法：
- `addEleValueByJson()` — 新增要素值集
- `updateEleValueByJson()` — 更新要素值集
- `delete()` — 删除要素值集
- `getByIdAndTableNameOrCode()` — 按 ID 查询要素值集

---

## 变更明细

### 变更文件

**文件路径**: `element-server/src/main/java/grp/pt/controller/basedata/ProcessElementController.java`

### 依赖注入变更

| 变更项 | 变更前 | 变更后 |
|--------|--------|--------|
| Controller 依赖 | `@Autowired ElementValueController elementValueController` | **已移除** |
| Service 依赖 | 无 | `@Autowired IElementValueService elementValueService` |
| 日志工具 | 无 | `Logger log = LoggerFactory.getLogger(ProcessElementController.class)` |

### 方法变更

#### 1. `approval()` 方法（私有方法，管理员审核核心逻辑）

| 调用点 | 变更前 | 变更后 |
|--------|--------|--------|
| 新增要素值集 | `elementValueController.addEleValueByJson(eleCode, elementDTO).getData()` | `doAddEleValue(eleCode, elementDTO)` |
| 更新要素值集 | `elementValueController.updateEleValueByJson(eleCode, elementDTO)` | `doUpdateEleValue(eleCode, elementDTO)` |
| 删除要素值集 | `elementValueController.delete(eleCode, busi_id)` | `elementValueService.delete(eleCode, busi_id)` |

#### 2. `deleteEleValue()` 方法（批量删除值集并审核上报）

| 调用点 | 变更前 | 变更后 |
|--------|--------|--------|
| 查询要素值集 | `elementValueController.getByIdAndTableNameOrCode(id, ele_source, ele_code).getData()` | `elementValueService.getByIdAndTableNameOrCode(id, ele_source, ele_code)` |

### 新增私有方法

#### `doAddEleValue(String eleCode, ElementDTO elementDTO)`

- **用途**: 审核通过后通过 Service 层添加要素值集
- **行数**: 约 30 行
- **逻辑说明**: 复用原 `ElementValueController.addEleValueByJson()` 中的核心业务逻辑，包含：
  - Session 获取与租户 ID 判断
  - 要素扩展类型判断（`eleExtendType == 1` 时 `tenantId = 0`）
  - 租户 ID 赋值策略（模板租户 / 当前租户 / 合并拆分场景）
  - 年度默认值设置
  - 调用 `elementValueService.addFeign()` 完成持久化

#### `doUpdateEleValue(String eleCode, ElementDTO elementDTO)`

- **用途**: 审核通过后通过 Service 层更新要素值集
- **行数**: 约 28 行
- **逻辑说明**: 复用原 `ElementValueController.updateEleValueByJson()` 中的核心业务逻辑，包含：
  - 合并场景（`isMerge`）：设置操作类型为 MERGE，处理合并列，删除被合并项
  - 拆分场景（`isSplit`）：设置操作类型为 SPLIT，清空 ID 后新增
  - 普通更新：调用 `elementValueService.updateFeign()`

---

## 修改原则

- 不改变任何 HTTP 接口的 URL、HTTP 方法、入参、出参
- 将 Controller→Controller 调用链替换为 Controller→Service 调用链
- 业务逻辑通过私有方法封装，保持行为完全一致
- 所有变更均添加 AI 代码标记

---

## 风险评估

| 风险项 | 评估 |
|--------|------|
| 接口兼容性 | 无影响，HTTP 接口完全不变 |
| 业务逻辑一致性 | 私有方法完全复用原 Controller 逻辑，行为一致 |
| 依赖关系 | Controller 层不再有同层依赖，符合架构规范 |
| 回归范围 | 仅影响流程要素模块的审核通过、批量删除场景 |
