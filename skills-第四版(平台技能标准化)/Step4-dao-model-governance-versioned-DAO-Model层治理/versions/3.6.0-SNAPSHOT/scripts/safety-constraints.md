# 修复安全约束与核心原则

## 核心原则

| 编号 | 原则 | 说明 |
|------|------|------|
| C-01 | **只动目录和包路径，不改业务逻辑** | 仅修改 package 声明和 import 语句 |
| C-02 | **安全迁移** | 先读取原文件 → 在新位置创建文件 → 更新引用 → 删除原文件 |
| C-03 | **逐步执行** | 按优先级逐项修复，每完成一项向用户确认 |
| C-04 | **保持可编译** | 迁移后确保所有 import 路径和 package 声明正确 |

## 安全约束（红线）

执行修复前必须遵守以下约束，**任何一条违反都必须立即停止并报告**：

| 编号 | 约束 | 说明 |
|------|------|------|
| S-01 | **不修改类名** | 任何类名、方法签名、接口定义不得改变 |
| S-02 | **不修改 HTTP 接口** | URL、HTTP 方法不得改变 |
| S-03 | **不修改 Entity 包名** | Entity 类的包名不得改变（除非用户明确要求迁移到 dao/entity/） |
| S-04 | **不修改业务逻辑** | 任何业务逻辑代码不得改变 |
| S-05 | **必须用户确认** | 重构前必须将修复计划展示给用户并获得确认 |
| S-06 | **无文件冲突** | 目标目录不得存在同名文件，迁移前必须检查 |

## 允许修改的范围

- Java 文件的 `package` 声明行
- Java 文件的 `import` 语句行
- 目录结构（创建新目录、移动文件、删除空目录）
- MyBatis mapper XML 中的 `namespace`（因包路径变更）
- `@MapperScan` 注解的 `basePackages`（因包路径变更）
- Spring 配置中的 `@ComponentScan` 路径（如因包路径变更需要）

## 不允许修改的范围

- 任何 Java 方法体
- 任何类/接口/枚举的定义结构
- 任何 XML 配置文件内容（MyBatis mapper XML 的 namespace 除外）
- 任何前端代码
- 任何 pom.xml（除非目录迁移影响了模块结构）
