# 修复安全约束与核心原则

## 核心原则

| 编号 | 原则 | 说明 |
|------|------|------|
| C-01 | **只做命名和规范调整，不改业务逻辑** | 仅修改类名、属性名、HTTP 方法兼容性，不修改方法实现内容 |
| C-02 | **约束优先** | 标记为"约束限制"的项不修改（URL、JSON 序列化兼容） |
| C-03 | **用户确认** | 所有修改计划须获得用户确认后执行 |
| C-04 | **保持可编译** | 修改后确保所有引用路径正确 |

## 安全约束（红线）

执行修复前必须遵守以下约束，**任何一条违反都必须立即停止并报告**：

| 编号 | 约束 | 说明 |
|------|------|------|
| S-01 | **不修改 HTTP 接口 URL** | 除非用户明确要求，不得修改任何接口路径 |
| S-02 | **不修改 HTTP 方法** | 除兼容性增强（DELETE/PUT 添加 POST 兼容）外，不得变更 |
| S-03 | **不修改业务逻辑代码** | 任何方法体内容不得改变 |
| S-04 | **不修改 DTO/VO 属性名** | 除非配合 `@JsonProperty` 保持兼容 |
| S-05 | **必须用户确认** | 重构前必须将修复计划展示给用户并获得确认 |
| S-06 | **无文件冲突** | 目标目录不得存在同名文件，修改前必须检查 |

## 允许修改的范围

- Java 类名（同步修改文件名和所有引用）
- `@DeleteMapping` / `@PutMapping` 兼容增强
- Spring Bean 名称（显式指定以解决冲突）
- 类文件名（与类名保持一致）

## 不允许修改的范围

- 任何 Java 方法体
- 任何接口 URL 路径
- DTO/VO 属性名（影响 JSON 序列化兼容性）
- 接口参数名（影响前端传参兼容性）
- 任何前端代码
- 已发布的接口 URL 和 HTTP 方法
