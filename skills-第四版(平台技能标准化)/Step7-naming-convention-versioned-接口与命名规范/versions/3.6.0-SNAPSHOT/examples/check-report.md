## 排序规则

1. **各检查项内**：记录按文件路径字母序排列，同一文件内按行号升序
2. **约束限制项**：按检查编号 S5-01 → S5-06 排列
3. **修复建议**：按优先级 高 → 中 → 低 排列，同优先级按文件路径字母序
4. **PASS 项**：如果某个检查项下所有文件全部通过，该检查项仍需输出表头，但在表格下方标注"全部通过"

## 状态值规范

| 状态 | 含义 | 使用场景 |
|------|------|---------|
| PASS | 检查通过 | 符合规范 |
| WARN | 警告 | 不符合规范但非致命 |
| FAIL | 失败 | 严重问题（如 Bean 冲突） |

## 分类值规范

| 分类 | 含义 | 使用场景 |
|------|------|---------|
| 可修复 | 可安全自动修复 | 类名修正、注解兼容 |
| 约束限制 | 不建议自动修复 | URL 路径、DTO 属性名 |
| — | 不适用 | PASS 状态的项 |
