# 可修复 vs 约束限制分类表

## 分类总览

| 修复项 | 分类 | 说明 | 决策依据 |
|--------|------|------|---------|
| @DeleteMapping/@PutMapping 兼容 | **可修复** | 添加 POST 兼容，保留原 HTTP 方法 | 不影响前端调用 |
| 类名后缀修正 | **可修复** | 需评估 Bean 名称影响 | 同步更新引用即可 |
| 类名大驼峰修正 | **可修复** | 需同步修改文件名和引用 | 同步更新引用即可 |
| 类名不规范缩写修正 | **可修复** | Ctrl→Controller 等 | 同步更新引用即可 |
| Bean 命名冲突 | **视决策树** | 需按确定性决策树判定 | → [deterministic-rules.md](deterministic-rules.md) |
| URL 路径结构调整 | **约束限制** | 影响前端调用，不得修改 | 影响外部系统 |
| HTTP 方法变更（GET/POST） | **约束限制** | 影响前端调用，不得修改 | 影响外部系统 |
| @PatchMapping | **约束限制** | 仅报告不修复 | 需要特殊评估 |
| DTO/VO 属性下划线→驼峰 | **约束限制** | 影响 JSON 序列化兼容性 | 影响前端解析 |
| 接口参数下划线→驼峰 | **约束限制** | 影响前端传参兼容性 | 影响前端调用 |
| 分页参数统一 | **约束限制** | 影响前端传参 | 影响前端调用 |
| 返回类型包装 | **约束限制** | 影响前端解析逻辑 | 影响前端解析 |
| 布尔属性 is 前缀 | **约束限制** | 影响 getter/setter 命名 | 影响序列化 |
| ID 后缀格式 | **约束限制** | 影响 getter/setter 命名 | 影响序列化 |
| 接口参数校验注解 | **约束限制** | 添加注解可能改变行为 | 可能影响接口行为 |
| 非 DTO/VO 层属性下划线命名 | **可修复** | 内部代码不影响序列化 | 仅内部影响 |
| 常量未全大写 | **可修复** | static final 字段命名 | 仅内部影响 |
| 单字符变量名 | **可修复** | 不影响外部 | 仅内部影响 |

## 可修复项详细说明

### 接口路径兼容修复（S5-01 相关）

| 操作 | 影响范围 | 风险等级 | 幂等性 |
|------|---------|---------|--------|
| DELETE/PUT 添加 POST 兼容 | 注解变更 + import 更新 | 低 | 已有 POST 兼容则跳过 |

### 命名修正（S5-02 相关）

| 操作 | 影响范围 | 风险等级 | 幂等性 |
|------|---------|---------|--------|
| 类名后缀修正 | 文件名 + import + Bean 名 + 注入点 | 中 | 后缀已合规则跳过 |
| 大驼峰修正 | 文件名 + import + Bean 名 + 注入点 | 中 | 已大驼峰则跳过 |
| 不规范缩写替换 | 文件名 + import + Bean 名 + 注入点 | 中 | 无黑名单缩写则跳过 |

### Bean 冲突处理（S5-06 相关）

| 操作 | 影响范围 | 风险等级 | 幂等性 |
|------|---------|---------|--------|
| 显式指定 Bean 名 | 注解参数 | 低 | 已无冲突则跳过 |
| 类重命名 | 文件名 + import + Bean 名 + 注入点 | 中 | 已无 "2" 后缀则跳过 |

## 约束限制项处理方式

约束限制项在检查报告中**标注但不修复**，处理方式：

1. 在报告的"约束限制项汇总"章节中列出
2. 说明约束原因
3. 如用户明确要求修复，需额外评估影响范围并制定兼容方案
4. 需要用户额外确认后方可执行

## 分类决策流程

```
INPUT: 一个检查发现的问题

├─ 修改是否影响外部系统调用？（前端、RPC、Feign）
│   ├─ 是 → 约束限制
│   └─ 否 → 继续判断
│
├─ 修改是否影响 JSON 序列化/反序列化？
│   ├─ 是 → 约束限制
│   └─ 否 → 继续判断
│
├─ 修改是否影响 HTTP 接口契约？（URL、方法、参数名）
│   ├─ 是 → 约束限制（DELETE/PUT 兼容增强除外）
│   └─ 否 → 继续判断
│
├─ 修改是否可能引入运行时异常？
│   ├─ 是 → 约束限制
│   └─ 否 → 可修复
│
└─ 结论：可修复
```
