# Step8 公共模块提取 - 检查规则

## 检查规则定义

### S8-01: util/ 包归属检查

**检查目标**: 工具类是否仍在业务模块（如 element-service）的 `util/` 包下

**检查方法**:
1. 扫描能力层模块 `src/main/java/**/util/` 目录
2. 列出所有 `.java` 文件
3. 分析每个工具类的引用关系

**判定标准**:
- 不依赖本模块 Service/DAO 层的纯工具类 → **推荐提取**
- 依赖本模块 Service 但被多模块引用的工具类 → **需人工判断**
- 仅在当前模块内部使用且强依赖 Service 的工具类 → **建议保留**

**输出格式**: 参见 examples/check-report.md

---

### S8-02: cache/ 包归属检查

**检查目标**: 缓存类是否仍在业务模块的 `cache/` 包下

**检查方法**:
1. 扫描能力层模块 `src/main/java/**/cache/` 目录
2. 列出所有 `.java` 文件
3. 分析缓存类是否依赖本模块的 DAO/Service

**判定标准**:
- 纯缓存工具/管理类（如 CacheManager、CacheUtil） → **推荐提取**
- 缓存加载器（依赖 DAO 查询数据） → **需人工判断**（可能需保留在 service 模块）
- 与特定业务强耦合的缓存类 → **建议保留**

---

### S8-03: constant/ 包归属检查

**检查目标**: 常量类是否仍在业务模块的 `constant/` 包下

**检查方法**:
1. 扫描能力层模块 `src/main/java/**/constant/` 目录
2. 列出所有 `.java` 文件

**判定标准**:
- 常量类通常无外部依赖 → **推荐提取**（绝大多数场景）
- 如常量类引用了本模块特有的枚举或配置 → **需人工判断**

---

### S8-04: enums/ 包归属检查

**检查目标**: 枚举类是否仍在业务模块的 `enums/` 包下

**检查方法**:
1. 扫描能力层模块 `src/main/java/**/enums/` 目录
2. 列出所有 `.java` 文件

**判定标准**:
- 枚举类通常无外部依赖 → **推荐提取**（绝大多数场景）
- 如枚举内嵌复杂业务逻辑（调用 Service） → **建议保留**

---

### S8-05: exception/ 包归属检查

**检查目标**: 异常类是否仍在业务模块的 `exception/` 包下

**检查方法**:
1. 扫描能力层模块 `src/main/java/**/exception/` 目录
2. 列出所有 `.java` 文件

**判定标准**:
- 自定义业务异常类（如 BusinessException） → **推荐提取**
- 通用异常处理类 → **推荐提取**
- 异常类几乎总是无外部依赖，大概率全部提取

---

### S8-06: config/ 包归属检查

**检查目标**: 配置类是否仍在业务模块的 `config/` 包下

**检查方法**:
1. 扫描能力层模块 `src/main/java/**/config/` 目录
2. 列出所有 `.java` 文件
3. **重点检查**: `@ComponentScan`、`@MapperScan`、`@EnableXxx` 等注解的扫描路径

**判定标准**:
- 通用配置类（如 Swagger、CORS、序列化配置） → **推荐提取**
- 包含 `@ComponentScan` 且扫描路径包含本模块特定包的 → **需人工判断**
- 包含 `@MapperScan` 的 → **建议保留**（与 DAO 层强耦合）
- 包含 `@Autowired` 注入本模块 Service 的 → **建议保留**

---

## 检查报告汇总规则

检查完成后，按以下规则汇总：

1. **推荐提取**: 无外部依赖或被多模块引用的公共类
2. **需人工判断**: 存在跨层依赖但可能值得提取的类
3. **建议保留**: 与业务模块强耦合，移动风险大于收益

每项输出: 文件路径 + 引用数量 + 依赖分析 + 提取建议
